# 4. Decisions Made

- Test cases are planned one-to-one for each FR/NFR as the baseline traceability spine.
- OCR containment receives additional boundary-specific tests because OCR remains the primary probabilistic subsystem risk.
- Synthetic fixtures are the default committed test data path.
- Restricted source PDFs, extracted full-text derivatives, OCR candidates, and substantive page text remain local-only and are not valid committed test fixtures.
- Implementation, packaging, and orchestration mappings remain pending until those lifecycle phases are authorized.
- Initial implementation uses standard C++ domain services because Qt is not installed in this environment. This preserves the approved backend-domain boundary and avoids coupling core validation to the presentation framework.
- The top-level `Makefile` delegates build/test execution to CMake/CTest and does not define package artifacts before Packaging phase approval.
- Public and internal method documentation should emphasize contracts, failure behavior, deterministic behavior, and source-text safety rather than restating syntax.
- Doxygen extensions should use standard commands only unless a repository Doxygen configuration explicitly supports custom aliases.
- Work-folder initialization uses caller-supplied source PDF metadata and page count; source PDF inspection remains inventory responsibility and is not performed by the initializer.
- Initial page text files are empty local artifacts; later extraction/review slices will populate or edit them through approved text services.
- Dependency capability detection in this slice reads filesystem metadata only and does not execute Poppler, Tesseract, OCRmyPDF, or any other external tool.
- Tool version reporting is represented as caller-supplied metadata until a later process-adapter slice safely probes tool versions.
- External modification detection uses file metadata and local content hashes as stale-write tokens; it does not provide collaborative locking and does not expose file contents in messages.
- Reviewed page text save uses the stale-write snapshot from load and rejects conflicts unless explicit overwrite is requested.
- Reviewed page text load/save returns page text only to the local caller; safe messages and test diagnostics avoid substantive page text.
- Review transition rules are deterministic and keep OCR/source-candidate labels out of accepted review status flow.
- Release eligibility remains accepted-only by default, preserving OCR-BND-001 containment.
- Readiness summaries aggregate only metadata counts from review-state and page-quality artifacts; they do not read reviewed page text files.
- Readiness safe messages omit page text and source PDF text.
- Validation repair planning is dry-run only in this slice; it proposes structural repair actions but does not modify artifacts.
- Repair planning safe messages omit page text and source PDF text.
