# 4. Decisions Made

- Output roots are validated as operating-system filesystem paths; URI-like values are rejected.
  - Related requirements: FR-029, NFR-001, NFR-008
  - Risk: Low; preserves local-only boundary and avoids false remote-service support.
- Protected backups copy `volume.json`, `review-state.json`, `page-quality.json`, and `pages/` only.
  - Related requirements: FR-023, NFR-007
  - Risk: Moderate; raw candidate backup remains out of this protection slice by design.
- Cleanup defaults to dry-run and requires confirmation for destructive execution.
  - Related requirements: FR-026, FR-030, NFR-007
  - Risk: Low; protects reviewed text from accidental removal.
- OCR routing schedules candidate generation only and never changes review status.
  - Related requirements: FR-014, FR-018, OCR-BND-001
  - Risk: Low; maintains probabilistic boundary containment.
- Volume metadata editing is model-based and rewrites complete `volume.json` artifacts.
  - Related requirements: FR-005 through FR-010
  - Risk: Moderate; future read/modify/write preservation may need richer JSON serialization.

- OCR routing remains review-gated and does not modify reviewed text, review status, or release eligibility.
