Working positions recorded for later Requirements review:

- Extractor initializes all `volume.json` fields.
- User modification of initialized metadata is expected.
- Collection grouping belongs at corpus level when available.
- Source PDF paths should be corpus-relative.
- `pageMap` remains in `volume.json` unless future evidence shows that larger volumes or richer page metadata make that inadequate.

- The product expectation is a native local application, not a browser-based application.
- Native application rationale is tied to reducing friction for the proposed local-file-centered workflow.
- Native framework and UI architecture remain undecided until later lifecycle phases.
